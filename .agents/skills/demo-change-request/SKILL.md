---
name: demo-change-request
description: Use for visible, low-risk, end-to-end changes that must be understandable in a live demo or stakeholder walkthrough.
---

Handle live changes in a controlled way.

Sequence:
1. restate the requested behavior change
2. identify impacted layers and contracts
3. propose a tiny plan
4. implement the smallest coherent vertical slice
5. update tests and docs
6. run the relevant quality gates
7. prepare a concise summary

Good uses:
- new KPI or metric
- new dashboard control
- small API contract extension
- tracking/logging feature with UI impact
